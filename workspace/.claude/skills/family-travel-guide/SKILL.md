## 核心定位

这不是一次性做一本手册，也不是AI生成的资料，而是一套**可复用的设计方法论**和**旅行任务系统**：

```
输入：景点名称 + 孩子年龄段
输出：12-20页 A4 纵向可打印 HTML 旅行任务书（含Base64内嵌图片），页数根据内容量动态调整
```

**产品定位**：像正式出版的儿童研学读物，像收费亲子旅行产品。

**设计哲学**: 从"PPT思维"切换到"儿童百科思维"，从"知识手册"升级为"探索任务书"。

```
PPT思维：一个知识点 = 一个模块 = 一个盒子
儿童百科思维：多样化呈现 = 丰富阅读体验
```

**设计基调**: DK儿童百科 × National Geographic Kids × 博物杂志。

关键词：`Information Design Educational Visual Storytelling Premium`

整体感觉：探索、变化、留白、高级、不花哨。

## V4.6 五项核心优化（执行铁律）

### 一、内容优化：降低知识堆砌感

**铁律：每页30秒内让孩子理解重点**

#### 1.1 避免知识罗列

检查规则：
- ❌ 禁止连续出现大量知识点罗列
- ❌ 禁止百科式介绍（如"经历了北凉、北魏、西魏、北周、隋、初唐、盛唐、中唐、晚唐"）
- ✅ 每个知识模块只保留核心内容

**三原则**：
```
少讲一点
讲有趣一点
讲具体一点
```

**示例转换**：

| 错误示范 | 正确示范 |
|---------|---------|
| "飞天经历了北凉、北魏、西魏、北周、隋、初唐、盛唐、中唐、晚唐的发展" | "飞天小时候胖胖的，到了盛唐变成了最漂亮的样子。" |
| "莫高窟现存洞窟735个，壁画4.5万平方米，彩塑2415身" | "莫高窟有700多个洞窟，如果一天看一个，要花两年才能看完！" |

#### 1.2 提高故事张力

重点页面必须采用**故事结构**：
- 九色鹿
- 藏经洞
- 常书鸿（或其他历史人物）
- 飞天

**故事模板**：`人物 → 冲突 → 选择 → 结果`

让孩子产生：好奇、惊讶、感动

#### 1.3 每页一个记忆点

**铁律**：每页必须有：
- 一个核心知识点
- 一个核心记忆点

孩子翻完后必须能记住一句话。

**示例**：
```
❌ 错误：飞天是佛教艺术，表现天宫乐舞，有多种姿态...
✅ 正确：飞天为什么没有翅膀？因为中国画家用飘带代替翅膀。
```

### 二、版式优化：视觉呼吸感与节奏

#### 2.1 内容预算制（V4.7新增，P0优先级）

**根因**：A4固定尺寸页面（`height:297mm; overflow:hidden`），内容塞太多就被裁切。

**铁律**：`单页总高度不超过A4可打印区域（约260mm，含边距）`

**内容预算规则**：

| 页面类型 | 字数上限 | 图片高度上限 | 总高度预算 |
|---------|---------|-------------|-----------|
| 封面（P1） | 50字 | 满屏 | 297mm |
| 目录（P2） | 400字 | 60mm | 260mm |
| 知识页（P3/P6/P9） | 350字 | 180mm | 260mm |
| 大数字页（P4） | 200字 | 80mm | 260mm |
| 九宫格页（P7） | 300字 | 200mm | 260mm |
| 漫画页（P8） | 250字 | 160mm×3 | 260mm |
| 涂色互动页（P14） | 200字 | 200mm | 260mm |
| 测验页（P15） | 300字 | 0mm | 260mm |
| 家长指南（P17） | 800字 | 0mm | 260mm |

**高度计算方法**：
```
标题区：约40-60px（含留白）
图片区：max-height按上述上限
文字区：字数 × 行高1.85 × 字号14px ≈ 字数 × 26px
互动区：约60-100px
IP区：约40-60px
```

**超预算处理规则**：
1. **优先缩减字号**：从14px降至13px或12px
2. **缩小图片**：降低max-height
3. **减少间距**：从32px降至24px或20px
4. **最后手段**：拆分到下一页

**CSS开发调试建议**：
```css
/* 开发阶段：设置overflow:visible查看内容是否溢出 */
.page {
  overflow: visible;  /* 开发调试用 */
}

/* 交付前：改回hidden */
.page {
  overflow: hidden;  /* 交付版本 */
}
```

**内容预算执行检查（V4.9.1新增，P1优先级）**：

> **铁律：生成内容后，必须计算实际高度是否达到预算的80%以上，避免页面下半空白**

**计算方法**：
```
实际高度 = 标题区(40-60px) + 图片区 + 文字区(字数×26px) + 互动区(60-100px)
预算高度 = 260mm（约980px）
```

**处理规则**：
```
如果 实际高度 < 预算高度×80%：
  → 优先增加内容到字数上限
  → 或增加底部互动元素
  → 或添加装饰纹样填充
  → 最后调整图片/间距比例
```

**区分"设计性留白"与"问题性留白"**：

| 页面类型 | 留白特性 | 处理方式 |
|---------|---------|---------|
| 整版大图页（模板1） | 满版图片+底部标题，留白是设计语言 | ✅ 不处理 - 这是"震撼冲击"的视觉特点 |
| 大数字页（模板2） | "大量留白"本身就是设计特点 | ✅ 不处理 - 留白让数字更突出 |
| 上下结构页（模板3） | 上图下文，文字少则底部空 | ⚠️ 需检查执行预算 |
| 九宫格页（模板5） | 网格布局，元素少则空 | ⚠️ 需检查执行预算 |
| 测验页/打卡页 | 纯文字/列表，内容少则空 | ⚠️ 需检查执行预算 |

**目标**：像高质量儿童出版物。

建立**四种统一模块**，颜色、图标、边框风格明显不同：

### 三、图片适配优化（V4.6强制）

#### 3.0 AI生图通用规则（V4.9新增，P0优先级强制）

> **铁律：所有AI生图必须遵循以下规则，否则视为不合格**

**规则1：所有AI生图prompt必须包含以下后缀**

```
不要包含任何文字、标题、标签、字母、数字或水印
```

**适用范围**：无论封面、插图、漫画还是配图，一律不加文字，文字内容全部由HTML/CSS叠加。

**根因**：AI生图模型（DALL-E、Midjourney等）无论prompt怎么写"NO TEXT"，大概率仍会在图里塞文字、条形码、logo。这是生图模型的通病，无法通过prompt完全避免。

---

**规则2：多面板/多格子页面的图片（如漫画页、对比页）**

**实施要求**：
- 每个格子只生成1张图，只描绘1个场景
- prompt必须包含：`只画这一个场景：[具体场景描述]，不要画其他场景`
- 禁止在一张图中压缩多个场景

**根因**：如果生成1张合并的四格漫画图，然后用`object-position`硬裁，结果所有格子显示同一张图的不同区域，布局完全错乱。

**生图Prompt模板**（以3列grid为例）：
```
请生成3张独立的漫画分镜插图，每张对应一个场景：

场景1：[具体场景描述1]，只画这一个场景，不要画其他场景
场景2：[具体场景描述2]，只画这一个场景，不要画其他场景
场景3：[具体场景描述3]，只画这一个场景，不要画其他场景

要求：
- 每张图都不要包含任何文字、标题、标签、字母、数字或水印
- 生成3张独立的图片，不是1张大图裁切
- 每张图片尺寸一致（推荐1:1或4:3）
- 水彩手绘风格
- 每张图留有对话气泡空间
- PNG透明背景
```

#### 3.1 图片文字分离原则（V4.7新增，P0优先级）

**铁律**：封面图、全出血图只用作背景氛围图，**所有标题、副标题、徽章文字一律用CSS/HTML叠加层实现，绝不依赖图片自带文字**。

**根因**：AI生图模型（DALL-E、Midjourney等）无论prompt怎么写"NO TEXT"，大概率仍会在图里塞文字、条形码、logo。这是生图模型的通病，无法通过prompt完全避免。

**实施规则**：
1. **封面图（P1）**：
   - 图片只作为满版背景，不包含任何文字内容
   - 标题、副标题、徽章全部用HTML绝对定位叠加
   - CSS渐变遮罩用于增强文字可读性

2. **章节开篇页（P5/P12）**：
   - 图片只作为视觉冲击元素
   - 所有说明文字用HTML层实现

3. **生图Prompt强制约束**：
   ```
   封面图prompt必须包含：
   ABSOLUTELY NO TEXT, NO LETTERS, NO CHARACTERS,
   NO WATERMARK, NO BARCODE, NO LOGO, NO STAMPS
   ```

4. **交付前检查**：
   - 即使prompt写了NO TEXT，AI生图仍可能出文字
   - 必须在交付前检查图片，如发现文字则重新生成

#### 3.2 漫画页面使用 object-fit: cover

**漫画页（P8）特殊规则**：漫画分镜图片必须使用 `object-fit: cover`

**CSS规则**：
```css
漫画分镜图片 {
  object-fit: cover;  /* 填充格子，制造分镜效果 */
  width: 100%;
  height: 100%;
}
```

**原因**：漫画页面需要统一尺寸的分镜格子，用 cover 保证每个格子完全填充，形成连贯的漫画分镜效果。

#### 3.3 其他图片使用 object-fit: contain

以下图片类型禁止使用 `object-fit: cover`，必须使用 `contain`：
- 文物图片
- 壁画图片
- 人物图片

**CSS规则**：
```css
文物图片、壁画图片、人物图片 {
  object-fit: contain !important;  /* 完整展示 */
}

封面背景图 {
```

## 核心策略：混合图源（V4.0新增）

**根本矛盾**：博物馆亲子旅行探索手册需要视觉准确性，但纯AI生图无法精确还原真实文物。

**解决方案**：核心文物用真实照片，AI只生成虚构元素。

### 图源分类规则

| 内容类型 | 图源策略 | 原因 |
|---------|---------|------|
| **镇馆之宝/核心文物** | 📸 真实照片作为参考 → AI生成手绘风插图 | 避免版权问题，同时保证视觉准确性 |
| **场景想象图**（古蜀城池等） | 🎨 AI生成 | 本来没有实物，允许想象 |
| **装饰纹样**（云雷纹等） | 🖌️ SVG/CSS直接画 | 简单几何图形，无需AI |
| **互动元素**（涂色页） | 🎨 AI线稿 | 只需要轮廓，不要求精确 |

### 参考照片获取方式

1. **优先级**：景点官网 > 维基百科 > 图库搜索
2. **要求**：高清、正面、无水印
3. **用途**：仅作为AI生图的参考，不直接嵌入（避免版权问题）

### AI生图降级机制（V4.5优化）

**降级前检查**：
1. 检查image_generate能力是否可用
2. 尝试3次重试（不同prompt策略）
3. 记录失败原因

**降级规则**：

| 页面类型 | 降级策略 | 说明 |
|---------|---------|------|
| **P1封面大图** | ⛔ 禁止降级 | 必须有图，否则报错退出 |
| **P5/P12震撼页** | ⚠️ 最后手段 | 3次失败后用CSS渐变+大字提示，报错警告 |
| **P7九宫格** | ⚠️ 可降级 | 3次失败后用SVG几何图形替代，输出警告 |
| **P8漫画** | ⚠️ 可降级 | 3次失败后用纯对话框+emoji，输出警告 |
| **其他页** | ✅ 可降级 | 降级到纯文字版，说明原因 |

**降级输出格式**：

```html
<div class="image-fallback-notice" style="background:#FFF3E0;border-left:4px solid #FF9800;padding:12px;margin:16px 0;font-size:13px;color:#E65100;">
  ⚠️ 配图生成受限：{原因（如"AI生图服务暂时不可用"/"生图超时"/"图片质量不符合要求"）}
  <br><small style="color:#8B7860;">本页面采用简化版式，建议稍后重新生成完整版</small>
</div>
```

**重试策略**：
1. 第1次失败：简化prompt，去掉复杂修饰词
2. 第2次失败：使用更通用的prompt模板
3. 第3次失败：更换生图风格参数

**禁止事项**：
- ❌ 禁止静默降级（不输出任何提示）
- ❌ 禁止在P1封面降级后继续生成
- ❌ 禁止用emoji替代核心文物插图

### 图文一致性验证

生成图片后，用LLM做"图文一致性自检"：
- 文字描述"9只鸟"，图片必须有9只鸟
- 文字描述"青铜面具"，图片颜色必须是青铜绿
- 不匹配则自动重试或降级到纯文字版

**一图一格铁律（V4.7新增，P0优先级）**：

> **核心原则**：漫画网格有几列，就必须生成几张独立图片。**禁止生成1张合并大图再用CSS裁切成多格**。

**根因**：如果生成1张合并的四格漫画图，然后用`object-position`硬裁，结果所有格子显示同一张图的不同区域，布局完全错乱。

**实施规则**：
1. **生图阶段**：明确要求生成N张独立图片（N = grid列数）
2. **禁止合并图**：绝不接受"一张图包含所有场景"的生图结果
3. **图片清单法**：先确定每页需要几张图、每张图的用途，再逐张生成

## P0级别检查（全量检查）

### 15.1 数据数学验证（P0）

**目的**：防止"唐代279窟占总数一半以上"这类数学错误

```
检查项：
- [ ] 所有比例计算经过验证（如"占一半"需数学计算确认）
- [ ] 示例验证：唐代279窟 ÷ 总数735 = 38%，不是"一半以上"
- [ ] 所有数字经过验证（年代、数量、面积）
- [ ] 距离/时间数据合理
- [ ] 作者归属正确（诗词、作品）
```

**自查方法**：
```
遇到"占X%""一半以上""大多数"等表述时，必须：
1. 找出分子和分母
2. 计算精确比例
3. 验证表述是否准确
4. 不准确则改为"为历代最多"等精确表述
```

### 15.16 内容一致性自查（P0优先级）

> **铁律：生成完成后必须执行以下自查，缺一不可**

**1. 同一信息的多种表达形式必须互相核对一致**

如果同一份内容里，某个知识点既以列表/图示形式出现，又以口诀、顺口溜、表格、配文等另一种形式重复出现，生成后必须逐项比对两处的顺序/数值/名称是否完全一致。

**自查步骤**：
```
[ ] 找出所有重复出现的知识点
[ ] 列出每种知识点的所有出现位置
[ ] 逐项比对：顺序/数值/名称是否完全一致
[ ] 如有不一致，以更权威、更标准的顺序为准，同步修改另一处
```

---

**2. 涉及固定顺序/固定对应关系的传统文化知识点，先核实标准答案**

涉及传统文化中固定顺序/固定对应关系的知识点（如五行、八卦、二十四节气、天干地支、十二生肖等），生成前必须先核实标准答案，再填入内容。严禁凭语感即兴生成。

**必须核实标准答案的知识点**：

| 知识类型 | 标准答案来源 | 示例 |
|---------|-------------|------|
| 故宫脊兽顺序 | 官方资料/权威文献 | 龙→凤→狮子→天马→海马→狻猊→押鱼→獬豸→斗牛→行什 |
| 五行五色对应 | 《礼记》《尚书》等经典 | 木-青、火-赤、土-黄、金-白、水-黑 |
| 二十四节气顺序 | 农历历法标准 | 立春→雨水→惊蛰→春分→... |
| 八卦方位 | 《周易》标准 | 乾-西北、坤-西南等 |
| 十二生肖顺序 | 传统文化标准 | 鼠→牛→虎→兔→... |

---

**3. 使用专有名词作为标题时，必须核实标题下列出的每一项是否真的属于这个体系**

当使用"五行""八卦""阴阳""三纲五常"等专有名词体系作为标题时，必须先列出该体系官方包含的全部条目，再核对生成内容是否一一对应。严禁混入相邻但不同的体系。

**常见混用错误**：

| 标题体系 | 正确内容 | ❌ 常见混入 |
|---------|---------|-----------|
| 五行五色 | 木火土金水 × 青赤黄白黑 | ❌ 混入"天-祭祀-靛蓝"（另一套礼制颜色） |
| 五方五色 | 东南西北中 × 青赤白黑黄 | ❌ 混入"紫色"（非正统五方色） |
| 三纲五常 | 三纲×五常 | ❌ 混入"三从四德"（另一套体系） |
| 文房四宝 | 笔墨纸砚 | ❌ 混入"印"（非传统四宝） |

---

**4. 面向儿童的简化表述，标注简化边界**

为适应儿童认知而使用的简化表述（如"不用一颗钉子""考好就是状元"），可以保留通俗说法，但必须在旁注或备注中加限定说明，避免孩子产生绝对化的误解。

**常见简化表述及标注方式**：

| 简化表述 | 准确限定 | 标注方式 |
|---------|---------|---------|
| "不用一颗钉子" | "极少使用钉子" | 小字备注："* 古建筑主要用榫卯结构，极少使用钉子" |
| "考好就是状元" | "第一名称状元，其余称进士" | 小字备注："* 科举第一名是状元，其他还有榜眼、探花等" |
| "皇帝住这里" | "皇帝在此办公/举行大典" | 小字备注："* 皇帝平时住后宫，太和殿只举行大典" |
| "皇帝穿黄袍" | "重大场合穿黄色龙袍" | 小字备注："* 皇帝不是每天都穿黄袍，只在重大场合穿" |

**自查步骤**：
```
[ ] 找出所有简化表述（如"全都是""从来不用""一定"等绝对化表达）
[ ] 为每个简化表述添加准确限定的旁注
[ ] 旁注使用小字号、淡色、斜体等视觉区分
```

## 十六、混合图源策略（V4.0核心）

### 16.1 参考照片获取

**目的**：用联网搜索获取核心文物/镇馆之宝的真实照片作为参考

**步骤**：
- 优先级：官网 > 维基 > 图库
- 要求：高清、正面、无水印
- **用途**：仅作为AI生图的参考，不直接嵌入（避免版权问题）

### 16.2 AI生图（参考照片生成）

**策略**：
- 核心文物：参考真实照片，生成水彩手绘风插图
- IP导览角色：纯AI生成水彩手绘风
- 场景想象图：AI生成（古蜀城池等）
- 失败时用简单几何图形+文字兜底

### 16.3 图文一致性验证

**方法**：
- 生成后用LLM自检："图中元素是否与文字描述一致？"
- 不匹配则重试或降级

### 16.4 装饰元素

**策略**：
- 纹样（云雷纹等）：SVG/CSS直接画
- 不调用AI，保证稳定性

### 16.5 图片内嵌强制规则（V4.1新增，P0优先级）

**铁律：所有图片必须Base64内嵌，禁止外部URL引用**

| 要求 | 说明 |
|------|------|
| ✅ AI生成的图片 | 转Base64后内嵌到 `<img src="data:image/...">` |
| ✅ 装饰SVG | 直接内联到HTML |
| ❌ 禁止外部URL | `src="https://s.coze.cn/..."` 等外部引用全部禁止 |

**原因**: 打印时断网，外部URL会变成空白区

**实施方式**:
1. AI生图完成后，获取图片二进制数据
2. 转为Base64字符串：`data:image/png;base64,iVBORw0KGgo...`
3. 直接写入 `<img>` 标签的 src 属性

**降级机制**:
当图片无法内嵌时：
- 核心文物：改用CSS渐变/几何图形 + 文字说明
- 装饰元素：用纯CSS实现
- 禁止交付含外部URL的HTML

**内容一致性自查（V4.1新增，强制执行）**：

> **铁律：生成完成后必须执行以下自查，缺一不可**

**1. 同一信息的多种表达形式必须互相核对一致**

如果同一份内容里，某个知识点既以列表/图示形式出现，又以口诀、顺口溜、表格、配文等另一种形式重复出现，生成后必须逐项比对两处的顺序/数值/名称是否完全一致。

**自查步骤**：
```
[ ] 找出所有重复出现的知识点
[ ] 列出每种知识点的所有出现位置（如"脊兽"出现在P7九宫格、P7顺口溜、P8漫画）
[ ] 逐项比对：
    - 顺序是否一致（如脊兽：龙→凤→狮子→... 行什）
    - 数值是否一致（如门钉：9×9=81，不能一处写9×9另一处写8×8）
    - 名称是否一致（如"狻猊"不能一处写"狻猊"另一处写"狈猫"）
[ ] 如有不一致，以更权威、更标准的顺序为准，同步修改另一处
```

**常见问题示例**：
- 脊兽九宫格的顺序 vs 顺口溜的顺序不一致
- 朝代时间轴的年份 vs 正文中提及的年份不一致
- 文物数量的数字 vs 图表中的数字不一致

---

**2. 涉及固定顺序/固定对应关系的传统文化知识点，先核实标准答案**

涉及传统文化中固定顺序/固定对应关系的知识点（如五行、八卦、二十四节气、天干地支、十二生肖等），生成前必须先核实标准答案，再填入内容。严禁凭语感即兴生成。

**必须核实标准答案的知识点**：

| 知识类型 | 标准答案来源 | 示例 |
|---------|-------------|------|
| 故宫脊兽顺序 | 官方资料/权威文献 | 龙→凤→狮子→天马→海马→狻猊→押鱼→獬豸→斗牛→行什 |
| 五行五色对应 | 《礼记》《尚书》等经典 | 木-青、火-赤、土-黄、金-白、水-黑 |
| 二十四节气顺序 | 农历历法标准 | 立春→雨水→惊蛰→春分→... |
| 八卦方位 | 《周易》标准 | 乾-西北、坤-西南等 |
| 十二生肖顺序 | 传统文化标准 | 鼠→牛→虎→兔→... |

**自查步骤**：
```
[ ] 识别内容中是否涉及固定顺序/对应关系的知识点
[ ] 联网搜索或查阅权威资料，获取标准答案
[ ] 用标准答案去套生成口诀、配色、图表
[ ] 禁止分别独立生成两份内容（避免出现"两套标准"）
```

**3. 使用专有名词作为标题时，必须核实标题下列出的每一项是否真的属于这个体系**

当使用"五行""八卦""阴阳""三纲五常"等专有名词体系作为标题时，必须先列出该体系官方包含的全部条目，再核对生成内容是否一一对应。严禁混入相邻但不同的体系。

**常见混用错误**：

|---------|---------|-----------|

| 三纲五常 | 三纲×五常 | ❌ 混入"三从四德"（另一套体系） |
| 文房四宝 | 笔墨纸砚 | ❌ 混入"印"（非传统四宝） |

**自查步骤**：
```
[ ] 识别标题是否为专有名词体系（如"五行""八卦"）
[ ] 列出该体系官方包含的全部条目
[ ] 核对生成内容是否一一对应

```

**4. 面向儿童的简化表述，标注简化边界**

为适应儿童认知而使用的简化表述（如"不用一颗钉子""考好就是状元"），可以保留通俗说法，但必须在旁注或备注中加限定说明，避免孩子产生绝对化的误解。

**常见简化表述及标注方式**：

| 简化表述 | 准确限定 | 标注方式 |
|---------|---------|---------|

| "考好就是状元" | "第一名称状元，其余称进士" | 小字备注："* 科举第一名是状元，其他还有榜眼、探花等" |

**自查步骤**：
```
[ ] 找出所有简化表述（如"全都是""从来不用""一定"等绝对化表达）

```

### 涂色画高度约束（V4.7新增，P0优先级）

**根因**：涂色线稿图通常是3:4竖构图（如1728×2304），在页面内只设置`max-width:85%`但不限制`max-height`，导致图片过高，和上面的票根区域一起超出A4高度被裁切。

**CSS修复规则**：
```css
/* P14 涂色互动页 - 图片高度约束 */
.page.tpl-activity .coloring-area {
  padding: 16px 0;
  text-align: center;
}

.page.tpl-activity .coloring-area img {
  max-width: 85%;
  max-height: 200px;  /* V4.7新增：严格限制高度 */
  width: auto;
  object-fit: contain;
  display: block;
  margin: 0 auto;
}

/* 票根区域确保不被挤压 */
.page.tpl-activity .ticket-area {
  min-height: 80px;
  padding: 16px;
  margin-bottom: 20px;
}
```

**通用规则（内容页图片高度约束）**：

> **铁律（V4.7）**：内容页（tpl-knowledge/tpl-activity/tpl-grid等）内的所有图片必须设置`max-height`，确保与文字内容合计不超过A4可打印区域（约260mm）。

```css
/* 通用内容页图片高度上限 */
.tpl-knowledge img,

.tpl-comic img {

.tpl-fullimage .bg-img {
```
