# family-travel-guide: 亲子旅行探索手册生成器 V4.5

**描述**: 基于景区真实文化与视觉基因，为家长生成17页A4可打印亲子旅行探索手册。参考DK儿童百科、国家地理儿童版的设计理念，采用多样化版式、丰富阅读节奏、充足留白。**核心文物参考真实照片让AI生成手绘风插图，避免版权问题同时保证视觉准确性**。

**V4.1新增**: 图片强制Base64内嵌、测验答案分离、打卡地图方位示意、互动区尺寸规范。

**V4.2新增**: 内容一致性自查四规则（同一信息多重表达核对、传统文化知识点标准核实、专有名词体系边界、简化表述边界标注）。

**V4.3新增**: 手册名称改为"亲子旅行探索手册"、P17增加推荐阅读/观看模板、删除"连一连"互动项目。

**核心特性**: 6种页面模板、7种阅读节奏、多样化互动、信息图设计、充足留白、水彩手绘风、杂志化品质。

**触发词**: 旅行伴学、景点讲解、亲子旅行、带孩子玩、旅行攻略、家庭旅行、景点伴学、亲子游手册、亲子旅行探索手册、游学手册、旅行手册、游学伴学、亲子游、景点手册、伴学手册、游学指南、旅行指南、亲子游学、游学游、伴游手册、伴游指南、游伴手册、游伴指南。

## 核心策略：混合图源（V4.0新增）

**根本矛盾**：博物馆亲子旅行探索手册需要视觉准确性，但纯AI生图无法精确还原真实文物。

**解决方案**：核心文物用真实照片，AI只生成虚构元素。

### 图源分类规则

| 内容类型 | 图源策略 | 原因 |
|---------|---------|------|
| **镇馆之宝/核心文物** | 📸 真实照片作为参考 → AI生成手绘风插图 | 避免版权问题，同时保证视觉准确性 |
| **场景想象图**（古蜀城池等） | 🎨 AI生成 | 本来没有实物，允许想象 |
| **装饰纹样**（云雷纹等） | 🖌️ SVG/CSS直接画 | 简单几何图形，无需AI |
| **互动元素**（涂色页） | 🎨 AI线稿 | 只需要轮廓，不要求精确 |

### 参考照片获取方式

1. **优先级**：景点官网 > 维基百科 > 图库搜索
2. **要求**：高清、正面、无水印
3. **用途**：仅作为AI生图的参考，不直接嵌入（避免版权问题）

### AI生图降级机制（V4.5优化）

**降级前检查**：
1. 检查image_generate能力是否可用
2. 尝试3次重试（不同prompt策略）
3. 记录失败原因

**降级规则**：

| 页面类型 | 降级策略 | 说明 |
|---------|---------|------|
| **P1封面大图** | ⛔ 禁止降级 | 必须有图，否则报错退出 |
| **P5/P12震撼页** | ⚠️ 最后手段 | 3次失败后用CSS渐变+大字提示，报错警告 |
| **P7九宫格** | ⚠️ 可降级 | 3次失败后用SVG几何图形替代，输出警告 |
| **P8漫画** | ⚠️ 可降级 | 3次失败后用纯对话框+emoji，输出警告 |
| **其他页** | ✅ 可降级 | 降级到纯文字版，说明原因 |

**降级输出格式**：

```html
<div class="image-fallback-notice" style="background:#FFF3E0;border-left:4px solid #FF9800;padding:12px;margin:16px 0;font-size:13px;color:#E65100;">
  ⚠️ 配图生成受限：{原因（如"AI生图服务暂时不可用"/"生图超时"/"图片质量不符合要求"）}
  <br><small style="color:#8B7860;">本页面采用简化版式，建议稍后重新生成完整版</small>
</div>
```

**重试策略**：
1. 第1次失败：简化prompt，去掉复杂修饰词
2. 第2次失败：使用更通用的prompt模板
3. 第3次失败：更换生图风格参数

**禁止事项**：
- ❌ 禁止静默降级（不输出任何提示）
- ❌ 禁止在P1封面降级后继续生成
- ❌ 禁止用emoji替代核心文物插图

### 图文一致性验证

生成图片后，用LLM做"图文一致性自检"：
- 文字描述"9只鸟"，图片必须有9只鸟
- 文字描述"青铜面具"，图片颜色必须是青铜绿
- 不匹配则自动重试或降级到纯文字版

## 十四、完整工作流程（5阶段自动生成，V4.5.1优化）

**执行模式**：5个阶段连续执行，无需中间确认，一次性完成手册生成。

**输出格式**：优先输出PDF格式（如平台支持），否则输出HTML格式。

**铁律**：严格按照5个阶段工作，每阶段完成后标注"=== Phase X 完成 ==="后自动进入下一阶段。

---

### Phase 1: 内容策划

**目标**：分析景点核心IP，设计适合{age}岁的阅读故事线

**步骤**：
1. 联网检索景点官方网站、博物馆官方资料
2. 核实历史、年代、数字等关键信息
3. 提取视觉基因（色彩、纹样、IP导览员）
4. 设计阅读故事线

**输出**：
- core_ip：景点核心IP和亮点（1-3个）
- story_line：适合{age}岁的阅读故事线概述
- content_outline：内容大纲（主要知识点）

**标注**：`=== Phase 1 完成 ===`

---

### Phase 2: 页面规划

**目标**：规划16-18页结构，分配内容，设计互动点

**步骤**：
1. 规划页数（16-18页）
2. 将内容分配到每页（每页type/content/interactions）
3. 设计互动点（**必须包含至少3个体验式任务**）

**体验式互动要求**：
- 身体互动（摆动作、读诗）
- 现场观察（找颜色、数数量）
- 成就感设计（评分、证书）

**输出**：
- total_pages：总页数
- page_structure：每页结构（页码/模板/内容/互动）

**标注**：`=== Phase 2 完成 ===`

---

### Phase 3: 知识生成

**目标**：为每页生成具体内容，确保知识点准确且有深度

**步骤**：

### 15.16 内容一致性自查（P0优先级）

> **铁律：生成完成后必须执行以下自查，缺一不可**

**1. 同一信息的多种表达形式必须互相核对一致**

如果同一份内容里，某个知识点既以列表/图示形式出现，又以口诀、顺口溜、表格、配文等另一种形式重复出现，生成后必须逐项比对两处的顺序/数值/名称是否完全一致。

**自查步骤**：
```
[ ] 找出所有重复出现的知识点
[ ] 列出每种知识点的所有出现位置
[ ] 逐项比对：顺序/数值/名称是否完全一致
[ ] 如有不一致，以更权威、更标准的顺序为准，同步修改另一处
```

---

**2. 涉及固定顺序/固定对应关系的传统文化知识点，先核实标准答案**

涉及传统文化中固定顺序/固定对应关系的知识点（如五行、八卦、二十四节气、天干地支、十二生肖等），生成前必须先核实标准答案，再填入内容。严禁凭语感即兴生成。

**必须核实标准答案的知识点**：

| 知识类型 | 标准答案来源 | 示例 |
|---------|-------------|------|
| 故宫脊兽顺序 | 官方资料/权威文献 | 龙→凤→狮子→天马→海马→狻猊→押鱼→獬豸→斗牛→行什 |
| 五行五色对应 | 《礼记》《尚书》等经典 | 木-青、火-赤、土-黄、金-白、水-黑 |
| 二十四节气顺序 | 农历历法标准 | 立春→雨水→惊蛰→春分→... |
| 八卦方位 | 《周易》标准 | 乾-西北、坤-西南等 |
| 十二生肖顺序 | 传统文化标准 | 鼠→牛→虎→兔→... |

---

**3. 使用专有名词作为标题时，必须核实标题下列出的每一项是否真的属于这个体系**

当使用"五行""八卦""阴阳""三纲五常"等专有名词体系作为标题时，必须先列出该体系官方包含的全部条目，再核对生成内容是否一一对应。严禁混入相邻但不同的体系。

**常见混用错误**：

| 标题体系 | 正确内容 | ❌ 常见混入 |
|---------|---------|-----------|
| 五行五色 | 木火土金水 × 青赤黄白黑 | ❌ 混入"天-祭祀-靛蓝"（另一套礼制颜色） |
| 五方五色 | 东南西北中 × 青赤白黑黄 | ❌ 混入"紫色"（非正统五方色） |
| 三纲五常 | 三纲×五常 | ❌ 混入"三从四德"（另一套体系） |
| 文房四宝 | 笔墨纸砚 | ❌ 混入"印"（非传统四宝） |

---

**4. 面向儿童的简化表述，标注简化边界**

为适应儿童认知而使用的简化表述（如"不用一颗钉子""考好就是状元"），可以保留通俗说法，但必须在旁注或备注中加限定说明，避免孩子产生绝对化的误解。

**常见简化表述及标注方式**：

| 简化表述 | 准确限定 | 标注方式 |
|---------|---------|---------|
| "不用一颗钉子" | "极少使用钉子" | 小字备注："* 古建筑主要用榫卯结构，极少使用钉子" |
| "考好就是状元" | "第一名称状元，其余称进士" | 小字备注："* 科举第一名是状元，其他还有榜眼、探花等" |
| "皇帝住这里" | "皇帝在此办公/举行大典" | 小字备注："* 皇帝平时住后宫，太和殿只举行大典" |
| "皇帝穿黄袍" | "重大场合穿黄色龙袍" | 小字备注："* 皇帝不是每天都穿黄袍，只在重大场合穿" |

**自查步骤**：
```
[ ] 找出所有简化表述（如"全都是""从来不用""一定"等绝对化表达）
[ ] 为每个简化表述添加准确限定的旁注
[ ] 旁注使用小字号、淡色、斜体等视觉区分
```

| 要求 | 说明 |
|------|------|

**内容一致性自查（V4.1新增，强制执行）**：

> **铁律：生成完成后必须执行以下自查，缺一不可**

**1. 同一信息的多种表达形式必须互相核对一致**

如果同一份内容里，某个知识点既以列表/图示形式出现，又以口诀、顺口溜、表格、配文等另一种形式重复出现，生成后必须逐项比对两处的顺序/数值/名称是否完全一致。

**自查步骤**：
```
[ ] 找出所有重复出现的知识点
[ ] 列出每种知识点的所有出现位置（如"脊兽"出现在P7九宫格、P7顺口溜、P8漫画）
[ ] 逐项比对：
    - 顺序是否一致（如脊兽：龙→凤→狮子→... 行什）
    - 数值是否一致（如门钉：9×9=81，不能一处写9×9另一处写8×8）
    - 名称是否一致（如"狻猊"不能一处写"狻猊"另一处写"狈猫"）
[ ] 如有不一致，以更权威、更标准的顺序为准，同步修改另一处
```

**常见问题示例**：
- 脊兽九宫格的顺序 vs 顺口溜的顺序不一致
- 朝代时间轴的年份 vs 正文中提及的年份不一致
- 文物数量的数字 vs 图表中的数字不一致

---

**2. 涉及固定顺序/固定对应关系的传统文化知识点，先核实标准答案**

涉及传统文化中固定顺序/固定对应关系的知识点（如五行、八卦、二十四节气、天干地支、十二生肖等），生成前必须先核实标准答案，再填入内容。严禁凭语感即兴生成。

**必须核实标准答案的知识点**：

| 知识类型 | 标准答案来源 | 示例 |
|---------|-------------|------|
| 故宫脊兽顺序 | 官方资料/权威文献 | 龙→凤→狮子→天马→海马→狻猊→押鱼→獬豸→斗牛→行什 |
| 五行五色对应 | 《礼记》《尚书》等经典 | 木-青、火-赤、土-黄、金-白、水-黑 |
| 二十四节气顺序 | 农历历法标准 | 立春→雨水→惊蛰→春分→... |
| 八卦方位 | 《周易》标准 | 乾-西北、坤-西南等 |
| 十二生肖顺序 | 传统文化标准 | 鼠→牛→虎→兔→... |

**自查步骤**：
```
[ ] 识别内容中是否涉及固定顺序/对应关系的知识点
[ ] 联网搜索或查阅权威资料，获取标准答案
[ ] 用标准答案去套生成口诀、配色、图表
[ ] 禁止分别独立生成两份内容（避免出现"两套标准"）
```

---

**3. 使用专有名词作为标题时，必须核实标题下列出的每一项是否真的属于这个体系**

当使用"五行""八卦""阴阳""三纲五常"等专有名词体系作为标题时，必须先列出该体系官方包含的全部条目，再核对生成内容是否一一对应。严禁混入相邻但不同的体系。

**常见混用错误**：

| 标题体系 | 正确内容 | ❌ 常见混入 |
|---------|---------|-----------|
| 五行五色 | 木火土金水 × 青赤黄白黑 | ❌ 混入"天-祭祀-靛蓝"（另一套礼制颜色） |
| 五方五色 | 东南西北中 × 青赤白黑黄 | ❌ 混入"紫色"（非正统五方色） |
| 三纲五常 | 三纲×五常 | ❌ 混入"三从四德"（另一套体系） |
| 文房四宝 | 笔墨纸砚 | ❌ 混入"印"（非传统四宝） |

**自查步骤**：
```
[ ] 识别标题是否为专有名词体系（如"五行""八卦"）
[ ] 列出该体系官方包含的全部条目
[ ] 核对生成内容是否一一对应
[ ] 多退少补：不属于该体系的删除，缺少的补上
```

---

**4. 面向儿童的简化表述，标注简化边界**

为适应儿童认知而使用的简化表述（如"不用一颗钉子""考好就是状元"），可以保留通俗说法，但必须在旁注或备注中加限定说明，避免孩子产生绝对化的误解。

**常见简化表述及标注方式**：

| 简化表述 | 准确限定 | 标注方式 |
|---------|---------|---------|
| "不用一颗钉子" | "极少使用钉子" | 小字备注："* 古建筑主要用榫卯结构，极少使用钉子" |
| "考好就是状元" | "第一名称状元，其余称进士" | 小字备注："* 科举第一名是状元，其他还有榜眼、探花等" |
| "皇帝住这里" | "皇帝在此办公/举行大典" | 小字备注："* 皇帝平时住后宫，太和殿只举行大典" |
| "皇帝穿黄袍" | "重大场合穿黄色龙袍" | 小字备注："* 皇帝不是每天都穿黄袍，只在重大场合穿" |

**自查步骤**：
```
[ ] 找出所有简化表述（如"全都是""从来不用""一定"等绝对化表达）
[ ] 为每个简化表述添加准确限定的旁注
[ ] 旁注使用小字号、淡色、斜体等视觉区分
```

## 二十四、PDF 导出优化（V4.4新增，P1优先级）

### 使用 playwright 生成 PDF

```python
from playwright.sync_api import sync_playwright

def html_to_pdf(html_file, pdf_file):
    """将 HTML 转换为 A4 纵向 PDF"""
    with open(html_file, 'r', encoding='utf-8') as f:
        html_content = f.read()
    
    with sync_playwright() as p:
        browser = p.chromium.launch(headless=True)
        page = browser.new_page()
        page.set_content(html_content, wait_until='networkidle')
        
        page.pdf(
            path=pdf_file,
            format='A4',
            print_background=True,
            margin={'top': '0mm', 'bottom': '0mm', 'left': '0mm', 'right': '0mm'}
        )
        browser.close()
```

### PDF 质量检查清单

- [ ] 页面居中对齐（body有 width: fit-content + margin: 0 auto）
- [ ] 图片完整显示（object-fit: contain，无overflow: hidden裁剪）
- [ ] 漫画panel高度充足（min-height: 120px+）
- [ ] 涂色画符合景点主题（不是通用卡通图案）
- [ ] PDF分页正确（无内容被切断）
- [ ] 所有图片都有alt属性或说明文字
- [ ] Base64图片正常显示（无空白区域）
